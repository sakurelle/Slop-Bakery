# Веб-интерфейс хлебозавода

## Запуск PostgreSQL и web

1. Скопировать `configuration/.env.example` в `configuration/.env`.
2. При необходимости изменить `POSTGRES_*`, `WEB_PORT` и `APP_SECRET_KEY`.
3. Выполнить:

```bash
docker compose --env-file configuration/.env -f configuration/docker-compose.yml up --build
```

или использовать скрипт:

```bash
./scripts/start_web.sh
```

или

```powershell
.\scripts\start_web.ps1
```

## Как открыть интерфейс

После запуска открыть:

- `http://localhost:8000`

## Как проверить CRUD

### Клиенты

1. Войти под `admin`.
2. Открыть `/customers`.
3. Создать клиента.
4. Открыть карточку клиента.
5. Отредактировать запись или нажать `Deactivate`.

### Сырьё и поставки

1. Войти под `warehouse`.
2. Открыть `/materials` и добавить новое сырьё.
3. Открыть `/deliveries` и создать поставку.
4. На карточке поставки добавить позиции поставки.
5. Проверить изменения на странице `/material-stock`.

### Технологические карты

1. Войти под `technologist`.
2. Открыть `/tech-cards`.
3. Создать технологическую карту.
4. Добавить в неё строки рецептуры через `Add Recipe Item`.

### Заказы

1. Войти под `client` или `admin`.
2. Создать заказ на странице `/orders`.
3. Открыть карточку заказа.
4. Добавить позиции заказа.
5. Под `admin` можно изменить статус заказа.

### Производство

1. Войти под `technologist`.
2. Открыть `/production`.
3. Создать производственную партию.
4. Проверить появление записи на `/finished-stock`.

### Контроль качества

1. Войти под `quality`.
2. Открыть `/quality`.
3. Добавить проверку сырья или готовой продукции.
4. Проверить фильтрацию по типу и результату.

### Счета и отгрузки

1. Войти под `admin` для счетов или `warehouse` для отгрузок.
2. Создать счёт по заказу на `/invoices`.
3. Создать отгрузку по заказу на `/shipments`.
4. Добавить состав отгрузки в карточке отгрузки.

## Как проверить отчёты

1. Открыть `/reports`.
2. Задать период.
3. Проверить блоки:
   сырьё на складе;
   сырьё ниже минимального остатка;
   остатки готовой продукции;
   заказы за период;
   продажи за период;
   производство за период;
   контроль качества за период.

## Как проверить аудит

1. Войти под `admin`.
2. Открыть `/audit`.
3. Выполнить несколько операций в интерфейсе:
   создание клиента;
   редактирование продукции;
   создание заказа;
   создание записи контроля качества.
4. Вернуться в `/audit`.
5. Отфильтровать записи по пользователю, таблице, действию и дате.

Аудит формируется триггерами БД, а веб-приложение передаёт в сессию соединения `app.current_user_id` и `app.current_user_ip`, чтобы триггеры могли связать изменения с пользователем веб-интерфейса.
