# Аудит базы данных

## Назначение аудита

Аудит нужен для фиксации критичных изменений в БД хлебозавода и последующего контроля действий пользователей. Журнал позволяет определить, кто, когда и какие данные изменял.

## Структура audit_log

Таблица `audit_log` содержит:

- `audit_id` — первичный ключ записи аудита;
- `user_id` — пользователь, выполнивший действие;
- `action_type` — тип действия (`INSERT`, `UPDATE`, `DELETE`, `LOGIN`, `LOGOUT`);
- `table_name` — имя изменённой таблицы;
- `record_id` — идентификатор изменённой записи;
- `changed_at` — дата и время события;
- `old_data` — старые данные в формате `JSONB`;
- `new_data` — новые данные в формате `JSONB`;
- `ip_address` — IP-адрес источника операции;
- `success` — признак успешности операции.

## Таблицы, для которых включён аудит

Аудит включён для следующих таблиц:

- `customers`
- `suppliers`
- `raw_materials`
- `products`
- `tech_cards`
- `recipe_items`
- `customer_orders`
- `order_items`
- `production_batches`
- `quality_checks`
- `invoices`
- `shipments`
- `users`
- `roles`
- `user_roles`

## Журналируемые операции

Для перечисленных таблиц журналируются:

- `INSERT`
- `UPDATE`
- `DELETE`

Дополнительно предусмотрена функция регистрации событий:

- `LOGIN`
- `LOGOUT`

## Реализация

- Используется универсальная триггерная функция на `PL/pgSQL`.
- Снимки записей до и после изменения сохраняются в `JSONB`.
- Для идентификации инициатора применяются параметры сессии `app.current_user_id` и `app.current_user_ip`.
- Для событий входа и выхода используется отдельная функция `log_auth_event`.
